# Tire Shop iOS

Standalone Expo / React Native iOS app for Tire Force US floor operations. It
talks to the same NestJS API as the main Tire Shop system over JWT bearer auth,
including the two-step login flow.

## Run It

```bash
pnpm install
pnpm start
```

Then press `i` to open the iOS simulator, or scan the QR code with Expo Go.

The API server address is hardcoded in `src/lib/server.ts`:

```ts
export const SERVER_URL = 'https://awstire.tail263731.ts.net';
```

To point a dev session or installed build at another API host, edit that value
and restart or rebuild the app.

## Build An IPA

This repo includes an iOS-only manual GitHub Actions workflow:

```text
.github/workflows/build-ios.yml
```

It produces an unsigned `tire-shop.ipa`. Because it is unsigned, iOS will not
install it directly; re-sign it with Sideloadly, AltStore, an Apple Developer
ad-hoc profile, or TestFlight signing outside this repo.

To build locally on a Mac with Xcode and CocoaPods:

```bash
pnpm install
pnpm build:ipa
```

The local build writes to `~/ipa-share/tire-shop.ipa` by default. Override that
with:

```bash
IPA_SHARE_DIR=/some/path pnpm build:ipa
```

## What It Does

- Inventory lookup, SKU management, stock adjustments, and inventory counts.
- Sales, quotes, invoices, PDF sharing, payment collection, and Tap to Pay.
- Customers, customer detail, edit/new customer, and sale history.
- Purchasing containers, returns, work orders, approvals, activity, users,
  roles, API keys, settings, money, accounting, FET, and EOD views.

## Notes

- Auth token is stored in `expo-secure-store`; cached user and tab preferences
  live in `AsyncStorage`.
- The app mirrors API response types manually in `src/lib/api.ts`; there is no
  code generation.
- Native `ios/` output is generated by `expo prebuild` during local and CI
  builds and is intentionally not committed.
- Native module versions are pinned to the Expo SDK. After changing Expo native
  dependencies, run `pnpm exec expo install --fix`.
